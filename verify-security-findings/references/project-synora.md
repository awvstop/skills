# 项目专用规则 — Synora

## 何时自动叠加（无需用户声明）

执行 **verify-security-findings** 且已能确定**被验证代码所在仓库**（如用户给出 `repo_path`、当前工作区即目标仓库、或 finding 中文件路径可解析到仓库根）时，在 **Step 0** 中**自动检测**是否为 Synora 工作区；**命中则自动叠加**本文件规则，**不要求**用户说「Synora」或 `@` 本文件。

**强信号**（满足**任一条**即可认定 Synora 并叠加）：

- 仓库根目录名或 `repo_path` 路径片段中含 `synora`（不区分大小写）
- `git remote -v`（在仓库根执行）输出中含 `synora`
- 仓库根目录 `README.md` / `README` 标题或首段明确出现项目名 **Synora**
- 根目录 `package.json`（或同等元数据）的 `name` / `description` 中含 `synora`（不区分大小写）

**弱信号**（**单独不足以**认定；仅作辅证或与强信号并用）：

- 仅存在顶层 `evaluation/` 等目录（其它仓库也可能有）

**不叠加**：用户在对话中**明确否定**（如「不是 Synora」「通用仓库」）时，即使路径像 Synora 也不应用本文件。

**强制叠加**：用户 **@** 本文件或粘贴其内容时，视为要求在**当前验证上下文**中应用本规则（用于显式启用或复核）。

验证结论的 **【依据】** 中建议写一句：`已按 Synora 规则叠加（自动检测：<触发的强信号摘要>）`，便于审计追溯。

## evaluation/ 与外部攻击面

- `evaluation/` 目录代码默认视为**本地评测工具链**，不属于外部用户直接攻击面。
- 对 `evaluation/` 相关 finding，若无「线上服务 / API / 后台任务**直接调用** evaluation 代码」的确凿证据，默认判 **`❌ FALSE POSITIVE`**（非外部攻击面）。
- **仍须已读**该 finding 涉及的**具体文件路径**（或等价片段），确认归属 `evaluation/` 且无线上调用线索后，方可适用本默认 FP；**禁止**仅凭路径名像 `evaluation/` 而未打开文件即结案。
- 仅在存在**明确线上调用证据**时，才按常规 Step 1～6 继续验证与定级。

## Sandbox 环境与严重度（Synora）

**结论先说**：在**可证明**「仅运行在强隔离、无高价值资产、与生产 API 不共用同一 Sink 路径」的 **sandbox / 评测 / CI 任务**里，**目录穿越、任意读等经典漏洞对外部攻击者往往不构成 `✅确认` 级影响**——可以**降级**、标 **`💡已知风险接受`**，或在**外部攻击者模型**下判 **`❌ FALSE POSITIVE`**，但**不能不经核实就一律当没问题**。

**为何不能一刀切「沙箱里都不算漏洞」**

- **同一代码多入口**：若与线上服务**共用**库函数 / 工具类，sandbox 里的 Sink 可能在生产路径上**同样可达**，须按常规 Step 1～4 追到**所有调用方**再定级。
- **沙箱≠无价值**：沙箱若挂载**宿主机路径**、可访**内网/元数据**、持**云凭证/数据集**，目录穿越或 SSRF 仍可能**实质危害**，不得因标签带 sandbox 就降级。
- **隔离强度**：容器默认逃逸面、挂载读写、特权模式等会削弱「纯沙箱」假设；证据不足时标 **`⏸️存疑`**，写明缺什么（隔离策略、挂载列表、网络策略）。

**可接受的降级/对外模型 FP 的典型条件（须同时尽量满足，并在结论里写清依据）**

1. **入口**：仅能通过**本地评测 Runner / 指定 CI Job / 无公网暴露**的批处理触发，且与对外 API **无共享路由或共享业务入口**（Step 1 证据）。
2. **边界**：穿越/读取范围**限制在沙箱工作区内**，无证据表明可读到**宿主机敏感路径、其他租户数据、生产卷**（Step 2/4/5）。
3. **资产**：该执行环境**不包含**生产密钥、用户 PII、可横向移动的内网凭据；若有则按实际影响定级，不按「沙箱」免责。
4. **复用**：已排除（或用户确认）该缺陷模式在**非 sandbox 代码路径**中的复用。

**与主 Skill 的关系**：本节约束**不改变**全局 CONTRACT 里「外部用户可触发才 ✅确认」的门槛，也**不免除**主 SKILL 中的**源码最低阅读集**（沙箱内判 FP/降级仍须基于**已读**相关路径代码）。仅在 Synora 场景下，对**已证实的纯沙箱路径**给出**更贴合实际修复优先级**的判定（常见为 **LOW / INFO / KNOWN RISK** 或对外部攻击面 **FP**），并在 **【依据】** 中明确写出 sandbox 边界假设。
