## Step 1 — 入口可达性按应用类型快速判断

| 应用类型 | 检查要点 |
|----------|----------|
| Web API | 路由注册表 + 中间件链 + 网关配置 |
| 桌面应用 | 监听地址(localhost?) + IPC机制 + CSWSH |
| 移动端 | deeplink/intent暴露面 + WebView桥接 |
| 微服务内部 | 是否仅限 service mesh 内部调用 |
| CLI工具 | 输入来源是否外部可控（参数/stdin/环境变量） |
| Serverless | 触发器类型(HTTP/事件) + API Gateway配置 |

## Step 2 — 信任边界层级定义

| 层级 | 描述 | 典型漏洞 |
|------|------|----------|
| L1 | 匿名用户 → 认证用户 | 认证绕过 |
| L2 | 普通用户A → 普通用户B | 水平越权 |
| L3 | 普通用户 → 管理员 | 垂直越权 |
| L4 | 客户端 → 服务端 | Client-Server 边界 |
| L5 | 外部服务 → 内部服务 | 网络边界 |
| L6 | 应用层 → 基础设施层 | 沙箱/容器逃逸 |
| L7 | 租户A → 租户B | 多租户隔离 |

## Step 3 — Source 可控性分级

| 级别 | 含义 | 典型证据 | 可利用性 |
|------|------|----------|----------|
| 完全可控 | 攻击者可直接注入请求体/参数/Header | req.body/query/params 直传 Sink | 高 |
| 部分可控 | 长度/字符集/格式受限但仍可构造 payload | 过滤后仍可注入关键字符 | 中 |
| 间接可控 | 经存储/转发/二次渲染后到达 Sink | 存储型 XSS、二次 SQLi、RAG 投毒 | 须追踪完整链 |
| 强约束可控 | 白名单/枚举/强类型后无法注入 | 仅枚举值、严格 schema | 不可控→FP |
| 不可控 | 来源为常量/配置/内部服务且不可伪造 | 非用户输入 | FP |

## Step 6 — 利用前提现实性

| 前提类型 | 判定 | 示例 |
|----------|------|------|
| 需管理员/高权限 | 若仅管理员可触达→降级或 FP | 需先登录 admin |
| 需内网/特定网络 | 外网不可达→降级并注明「内网可利用」 | 仅 10.x 可访问 |
| 需调试/开发配置 | 生产默认关闭→降级并注明条件 | DEBUG=true |
| 需多步组合 | 单步不可利用→标组合链并独立定级 | 先信息泄露再 SSRF |
| 默认配置下成立 | 常见部署即满足→维持原级 | 默认无认证 |

## Step 4 — 天然断点及其失效条件（必须检查）

| 天然断点 | 失效条件 |
|----------|----------|
| 类型转换（string→int/bool/enum） | 转换前已拼接进危险上下文；宽松类型语言的隐式转换 |
| ORM参数化 / Prepared Statement | `.raw()` / `.extra()` / 字符串拼接进 QuerySet；动态表名列名 |
| 模板引擎自动转义（Jinja2/React等） | `\|safe` 过滤器 / `Markup()` / `{% autoescape false %}` / `dangerouslySetInnerHTML` / `href="javascript:"` |
| 序列化/反序列化重构 | 自定义反序列化器 / 宽松的 JSON Schema / 缺少 additionalProperties: false |
| 框架级CSRF防护 | 特定 Content-Type 豁免 / `@csrf_exempt` / SameSite=None + 第三方上下文 |
| Content-Type强制校验 | 某些框架对空 Content-Type 的 fallback 行为；`text/plain` CORS 简单请求 |
| 路径规范化（realpath） | TOCTOU竞争 / 先拼接后规范化（顺序错误）/ 符号链接未处理 |
| 编译期类型安全（Rust/Go） | unsafe 块 / cgo 调用 / 反射 / 泛型 interface{} |
| 长度截断 / 字符集限制 | 截断后仍保留危险语义（SQL截断攻击）；多字节字符集绕过 |

## Step 5 — 按漏洞类别的验证要点

| 漏洞类别 | 关键验证点 |
|----------|-----------|
| **SQLi** | ORM vs raw query + 参数化是否贯穿所有路径 + 动态表名/列名/ORDER BY处理 |
| **XSS** | 输出上下文(HTML/JS/URL/CSS) + 模板引擎转义覆盖范围 + DOM-based场景 + CSP策略 |
| **SSRF** | 协议限制(仅http/https?) + 内网IP黑名单(含IPv6/短格式/DNS rebinding) + 重定向跟随 |
| **路径遍历** | 规范化顺序(先规范化后拼接?) + 根路径jail + 符号链接跟随 + 空字节截断(老版本) |
| **反序列化** | 反序列化类型(原生/JSON) + gadget chain可用性 + 类型白名单 + 签名校验 |
| **IDOR** | 资源标识符可预测性 + 对象级权限校验(非仅认证) + 批量枚举可行性 |
| **命令注入** | 调用方式(shell=True?) + 参数数组 vs 字符串拼接 + 环境变量注入 |
| **XXE** | XML解析器配置(外部实体/DTD是否禁用) + 文件协议 + OOB通道 |
| **认证绕过** | 时序攻击 + 密码重置流程 + JWT算法混淆(none/HS256-RS256) + 会话固定 |
| **业务逻辑** | 支付/状态机/权限是否仅前端或可被绕过；关键操作是否服务端二次校验 |
| **竞态/TOCTOU** | 检查→执行之间是否有竞态；并发下权限/余额/库存是否原子校验 |
| **不安全默认** | 默认密钥/调试开关/默认账号、生产是否仍用开发配置 |
| **AI/LLM/Prompt 注入** | 用户输入是否直入 system/user prompt；LLM 输出是否进入 SQL/命令/文件；Tool 是否有 scope/审批 |
| **Prototype Pollution** | 深合并/object.assign 是否接受用户 key；是否影响授权/模板/查询构造 |
| **Mass Assignment** | 是否白名单绑定；req.body 是否直传 Entity；whitelist/fields 是否覆盖敏感字段 |
| **GraphQL** | depth/cost/alias/batch 限制；字段级授权；嵌套 resolver 是否逐层校验 context.user |

## AI 扫描器常见误报模式

| 误报模式 | 快速判定方法 |
|----------|-------------|
| 忽略认证/授权边界 | 检查路由中间件是否强制认证 |
| 忽略框架自动防护 | 确认ORM参数化、模板自动转义等是否生效，并检查失效条件 |
| 内部函数当外部入口 | 追踪调用链确认有无外部可达路径 |
| 需先有高权限才能利用 | 评估是否为正常功能而非漏洞 |
| 默认配置下不触发 | 确认实际部署配置与报告假设是否一致 |
| 同一模式重复报告 | 确认根因后按防护栏规则批量处理 |
| 将设计行为标记为漏洞 | 对照产品需求确认是否为预期功能 |
| 忽略间接数据流 | 注意存储型攻击路径，AI扫描器可能仅追踪同步调用链 |
