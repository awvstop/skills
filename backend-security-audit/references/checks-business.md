# Business Logic / Sensitive Flows — BSAF v3.6

> 业务逻辑+端到端流程安全。幂等/状态机/时间窗/重放唯一定义于此。

## 通用业务逻辑

- **负数输入**：amount/price/quantity 无 >0 校验 → 负数购买=退款 🔴
- **金额精度**：float 非 Decimal/分 → 🟡
- **整数溢出**：极大 quantity/amount（如 2^31-1）→ Java int 溢出变负 🔴；JS Number > 2^53-1 → 精度截断 → 金额错误 🔴；须验证上限
- **批量操作**：无逐条所有权校验 → 🟡；无数量上限 → DoS 🟡
- **枚举攻击**：登录/注册/重置/MFA 区分「账号不存在」vs「密码错误」→ 账号枚举 🟡；响应时间差异同样泄露 → 须恒定时间响应
- **顺序 ID 枚举**：订单/发票/用户 ID 为自增整数 + 无所有权校验 → 遍历所有数据 🔴（结合 BOLA 检查）
- **礼品卡/优惠码暴力**：短码（6位数字）+ 无 rate limit → 穷举 🟠

## 电商/支付（检测到 payment/order/cart/checkout）

- 价格从 req.body.price 直接用于订单/支付 → 🔴（须后端重算）
- 优惠券多次应用/负折扣 → 🟡
- 退款金额>订单金额 → 🔴
- 订单状态非法跳转 → 🔴

## SaaS/多租户（检测到 tenant/org/workspace）

- 查询缺 tenantId/orgId → 跨租户泄露 🔴
- list/search/export 是否绑定租户过滤
- 管理员跨租户范围限制

## 敏感流程 Checklist

| 维度 | 缺失 → |
|------|--------|
| **Rate limit**（登录/注册/重置/OTP/支付） | 🟡/🔴 |
| **幂等**（支付/转账/通知有幂等 key 或去重） | 重放 🟡 |
| **状态机约束**（退款/发货/审批校验前置 status===PAID） | 状态绕过 🟡 |
| **顺序依赖**（多步流程强制顺序） | 跳步 🟡 |
| **重放**（敏感 mutation timestamp+nonce 或 token 一次性） | 🔴 |
| **异步补偿**（异步任务失败可重复→双花/重复扣款） | 🔴 |
| **并发双花**（余额/库存 读-改-写 有 transaction/锁） | 竞态 🔴 |
| **审批边界**（审批人≠申请人，不可绕过） | 🔴 |
| **时间窗**（限时活动仅前端控制时间） | 🟡 |
